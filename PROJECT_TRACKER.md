## Key Decisions
- (2026-04-26) **Strategy editor canvas wires to the flat schema** (ADR-10 + `STRATEGY_EDITOR_WIRING_PLAN.md`). The backend's `EntryRules` is a single flat `ConditionGroup{logic, conditions[]}` ‚Äî no nesting. The canvas's nested OR-group visual is therefore unrepresentable; it gets dropped, replaced with N rule nodes feeding one AND/OR gate. Save model is **explicit Save Draft only** (drawer Apply marks dirty; header Save Draft is the one PUT). Schema-less UI fields (Long/Short/Either direction, per-condition Timeframe, separate-tree Exit signal) are hidden ‚Äî shipping fields the backend can't honor is worse than shipping nothing. Indicator periods become constrained dropdowns (SMA `[20,50,200]`, EMA `[12,26]`, RSI static, others hidden) because the indicator vocabulary is enum-scoped, not parametric. Work splits into two PRs: PR-1 (round-trip existing rules) before PR-2 (CRUD + name edit + indicator picker). Lifts ADR-9 ¬ß2.
- (2026-04-26) **Phase 3.2b‚Äì3.5 ship safe writes only** (ADR-9). Wizard preset‚Üíentry_rules is starter conditions (refine in editor); editor canvas does NOT round-trip (only deploy/pause toggle persists); bot dashboard activity log is empty-state until backend grows a log endpoint; portfolio backend semantics (paper-trading) differ from journal UI semantics so the page stays on local CSV with the API surface ready for the next slice; backtest Re-run uses opinionated 12mo / 1M PKR range. Each deferral has an explicit lift trigger documented in ADR-9. The full type + route surface still ships so follow-ups don't re-derive contracts.
- (2026-04-26) **Server-side proxy routes (`app/api/*`) are the canonical pattern for client‚Üíbackend writes**. The browser never sees the backend JWT; route handlers mint it from the NextAuth session via `auth()` + `signBackendJwt()` and forward to FastAPI. Each handler has a small fixed shape: parse params, get JWT, call the typed wrapper, map `ApiError` to JSON with the same status. This decouples client code from auth and keeps the backend URL behind one env var (`NEXT_PUBLIC_API_BASE_URL`).
- (2026-04-26) **`/strategies` list ships without per-row enrichment** (ADR-8). The frontend's `Strategy` row shape includes `signals` (today), `bt` (latest backtest %), `sharpe`, and `outputs` glyphs; none are on the bare `StrategyResponse`. Sourcing them costs 2N+1 round trips for the index page of authoring work. List slice ships with empty sentinels; enrichment lifts when users with ‚â•10 strategies report it as a workflow problem, OR when the backend grows a `?include=summary` flag. `type` is hardcoded `"Custom"` (the frontend's mean-rev/momentum/trend taxonomy doesn't exist server-side and would mis-classify multi-indicator strategies). `universe` is derived locally from `stock_symbols` or `stock_filters.sectors`.
- (2026-04-26) **HS256 JWT minted with `node:crypto`, not `jose`** ‚Äî psx-ui's package.json keeps its hard-line minimalism (`next-auth` is the only new runtime dep). HS256 is HMAC-SHA256 + base64url; ~30 lines of stdlib code, edge-of-correctness-zero. `jose` is transitively present via `next-auth` but importing transitives couples us to `next-auth`'s internal choices. Server-only guard in `lib/api/jwt.ts` throws if loaded in the browser.
- (2026-04-26) **`/signals` page split: server component fetches, client island renders.** Server component reads session, mints JWT, calls backend, maps response, passes data into `<SignalsView initialSignals={‚Ä¶} />`. The client island (`signals-view.tsx`) keeps every byte of the existing inline-style markup unchanged. This is the canonical Next.js 16 pattern for protected data routes and avoids needing SWR or a client-side `/api/auth/token` endpoint for first paint. Refetch-on-focus and polling get added per-route only when their need is observed.
- (2026-04-25) **NextAuth v5 chosen over hand-rolled OAuth** (ADR-7). The 3-runtime-dep purity protects against UI bloat (Tailwind, shadcn, Radix, SWR), not auth infra. Two facts forced NextAuth: (1) the FastAPI backend already verifies NextAuth-issued HS256 JWTs and the live Neon DB has user rows keyed off `account.providerAccountId`, so hand-rolling would orphan accounts; (2) the previous frontend's 34-line integration ports verbatim. `next-auth@5.0.0-beta.31` is the only new runtime dep in the entire migration.
- (2026-04-25) **`proxy.ts`, not `middleware.ts`** ‚Äî Next.js 16 (`16.2.4`) renamed the file convention. The previous name still works but emits a deprecation warning at build time. Using the new name keeps the build clean and survives the eventual removal.
- (2026-04-25) **Pre-auth architecture documented as ADRs at `PRE_AUTH_DECISIONS.md`** (6 ADRs). Two have code shipped today; the other four fire when their trigger event arrives. Rationale: cost of writing 6 ADRs now ‚âà 20 min; cost of reversing wrong choices later = migration work, security incidents, compliance review.
- (2026-04-25) **No ESLint in the project** ‚Äî the `NEXT_PUBLIC_` leak guard is a 150-line `scripts/check-env.mjs` wired to `prebuild`, not an ESLint rule.
